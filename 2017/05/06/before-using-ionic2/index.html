IONIC FRAMEWORK2를 사용하기 전 TIP

2017-05-06 ionic

필자가 아이오닉 프레임워크를 사용하면서 얻은 것들을 나눠보려고 한다.

본 포스팅은 2017–02–08을 기준으로 제일 최신버전인 아이오닉프레임워크 2.0.0 Stable 버전 기준으로 작성됩니다.

왜 아이오닉 프레임워크를 사용하게 되었는가.

재직하고 있는 회사에서 기존에 운영하고 있는 서비스를 라이트하게 만들어서 글로벌 서비스 개발 착수를 해야한다고 통보받았다. 이유는 시장 확장도
있지만, IR 투자를 위해서 만드는 서비스라는게 큰 이유라고 한다.

팀은 서버 엔지니어와 프론트엔드 엔지니어 (나), 그리고 글로벌 비즈니스 담당자 3명이서 글로벌 서비스를 개발하기로 결정했다. 하지만 개발팀에
서비스 하나 제대로 만들 인력이 없는 상황이었고, 네이티브 애플리케이션 개발자도 없는 상황이라서 (있었지만 본사 일때문에 개발 할 여력이 되지
않았었나보다.) 대안을 찾아보던 중에 하이브리드 웹 애플리케이션으로 개발하는게 가장 좋은 대안이었다.

하이브리드 웹 애플리케이션을 어떻게 개발하면 좋을까 생각하다가, 네이티브 API를 그대로 쓸 수 있는 각종 라이브러리들을 패키징하고,
UI측면에서도 네이티브 컴포넌트와 비슷한 프레임워크가 내 눈에 들어왔었다. 바로 아이오닉
프레임워크였다.


IONIC FRAMEWORK 소개 페이지

사용할 프레임워크 선정 전에, 미리 예제 애플리케이션의 소스를 빌드하며 돌려보았다. 주로 봤던건 빌트인 컴포넌트가 어떤식으로 돌아갈지
궁금했는데, 마치 웹이 아닌냥, 네이티브 컴포넌트와 아주 비슷하게 동작했고, 애니메이션도 끊김없이 잘 작동했다는 것이다. 더욱 놀라웠던 사실은,
한 컴포넌트를 사용하여 빌드하고 각기 다른 기기에서 실행하면 그 기기의 디자인이 알아서 들어가는 것이다.


같은 소스로 빌드했으나, 서로 다른 플랫폼에서 맞춤 UI가 제공된다.

보고 정말 놀랬다. 각 플랫폼에 디자인 가이드라인도 맞춰주었고, 순정 컴포넌트의 특색을 그대로 살려서 빌드가 되었다. 아마 User-Agent
에 따라서 다른 스타일 시트를 사용하는 방식인가보다.

각설하면, 아이오닉 프레임워크가 각 플랫폼에 맞게 알아서 따로 빌드를 해주니, 개발자가 UI에 그렇게 큰 리소스를 투자하지 않고, 단순
비즈니스로직에 집중할 수 있어 생산성은 엄청나게 높다고 평가할 수 있다. 그래서 난 아이오닉 프레임워크를 사용하기로 결정했다.

개발을 시작하기 전에

우리가 사용할 아이오닉 프레임워크는 기본적으로 이런것들을 요구한다.

  • Angular2 : SPA(Single Page Application)를 개발할때 사용하는 프레임워크. 양방향 데이터 바인딩을 지원하여,
    DOM조작에 있어 쉽고 간편하다. 전 버전에 비해 2배가량 빨라졌다고 한다.
  • TypeScript : Angular2 가 기본적으로 TypeScript를 내세우면서 자연스럽게 TypeScript가 선정된 것 같다.
    기본적으로 기존의 자바스크립트와 비슷하다. 구글이 만들었던 AtScript도 있었지만 망하고 이미 기존의 Javascript로 트랜스 컴파일링
    되는 TypeScript 언어를 마이크로소프트에서 개발하고 있었기 때문에, 구글측에서도 TypeScript를 선정했다고 한다.
    (TypeScript를 사용하지 않고, ES7 을 사용할 수도 있다.)
  • Cordova : Apache 재단에서 지원중인 하이브리드 웹 애플리케이션을 개발할 수 있는 프레임워크이다.

또한 이런것들 말고도 Webpack, SASS 등 여러가지가 있지만, 배경지식에 대한 설명은 여기까지 하겠다.

본인은 한번도 Angular2 와 TypeScript 를 사용해 본적이 없었다. 데드라인이 짧기도 했고 (3주정도 된다.) 리소스 단축과 빠른
출시가 제일 중요했기 때문에 먼저 배경지식을 익히는게 제일 중요했다. 하루만에 배경지식의 튜토리얼을 학습했고, 부족했던 부분은 만들면서 배울 수
밖에 없었다. 아래는 본인이 학습할때 참고한 튜토리얼 및 강좌 이다.

  • Angular 2 In 60 Minutes 튜토리얼 영상
    (https://www.youtube.com/watch?v=-zW1zHqsdyc)
  • 송재호님의 타입스크립트 동영상 강좌
    (https://www.youtube.com/watch?v=EY1-5wDhrTk)
  • 아이오닉 프레임워크의 LearnAngular
    (http://learnangular2.com/)
  • 김훈민님의 웹 프론트엔드 개발자의 얕고 넓은 Rx
    이야기(https://www.youtube.com/watch?v=Yh2_xsr8mXQ)

사실 개인적인 의견은 아이오닉 프레임워크로 단순히 서비스를 빨리 개발하기 위해서 필요한 학습 내용은 Angular2의 양방향 데이터 바인딩,
컴포넌트만 익히면 바로 개발이 가능하다고 생각한다. (물론 나머진 만들면서 충분히 배울 수 있는 것들이고 나머진 옵셔널)

위는 HTTP 호출 후 Observable로 넘겨 사용하는 단에서 Subscribe 하는 방식의 코드인데, HTTP 모듈을 사용하면서
RX(Reactive X)를 사용하게 되었다. 꼭 필요하진 않지만, 개념정돈 알아둬야 감이 잡히고 잘 사용할 수 있게 된다. 또한 콜백함수에
대한 여러 해결책중 중요한 한가지로써 자바스크립트 개발자라면 한번쯤 접해보자. (위에 올려둔 김훈민님 강좌 강추)
